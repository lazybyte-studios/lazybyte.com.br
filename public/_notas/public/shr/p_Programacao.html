História

A história do HTML é uma jornada fascinante de crescimento, adaptação e inovação, desempenhando um papel crucial na evolução da World Wide Web.

  • 1989

    Tim Berners-Lee, cientista britânico, cria o HTML (Hypertext Markup Language) para facilitar a troca de informações entre cientistas no CERN (Organização Europeia para Pesquisa Nuclear).

  • 1991

    Lançamento do HTML 1.0 como especificação inicial.

  • 1995

    HTML 2.0 introduz formulários e tabelas.

  • 1997

    Popularização da web; surgimento de navegadores como Netscape Navigator e Internet Explorer (96) ; Guerra de navegadores e HTML 3.2 traz suporte a applets Java. Desenvolvimento para múltiplos navegadores. Uso de tabelas.

História

  • 1999

    O HTML 4.01 é publicado pelo W3C (World Wide Web Consortium). Introdução de CSS (Cascading Style Sheets) para controle de estilo.

  • 2000

    XHTML 1.0 combina XML e HTML. Aumento do rigor na sintaxe e na estrutura.

  • 2008

    Início do desenvolvimento do HTML5. Foco em melhorias de semântica, multimídia integrada (áudio e vídeo) e APIs.

  • 2014

    HTML5 é oficialmente recomendado pelo W3C. Adoção generalizada por desenvolvedores e navegadores.

História

  • 2019

    Mudança para um modelo "Living Standard". Atualizações contínuas, sem versões específicas.

  • Dias atuais...

    Uso onipresente em todas as páginas web. Integração com tecnologias como CSS e JavaScript para criar experiências interativas.

  • Futuro do HTML...

    Desenvolvimento contínuo para atender às demandas evolutivas da web. Exploração de novas funcionalidades e padrões.

Princiais áreas de um site

  • Cabeçalho (header): faixa na parte superior com um grande título e / ou logotipo.
  • Barra de Navegação: Links para as principais seções do site
  • Conteúdo: contém a maior parte do conteúdo.
  • Barra Lateral (sidebar): Algumas informações periféricas, links, cotações, anúncios etc.
  • Rodapé (footer): parte inferior da página que geralmente contém avisos de direitos autorais ou informações de contato.

Tipos de Tags

No HTML, as tags podem ser divididas em dois tipos quanto ao fechamento:

  • Tags de Abertura Única (Self-closing) ou órfãs:
    tags não têm uma tag de fechamento separada.
    <meta name="" content="" >

  • Tags com abertura e fechamento:
    tags que têm uma tag de abertura e uma de fechamento correspondente.
    O conteúdo está entre a tag de abertura e a tag de fechamento.
    <p> Conteúdo </p>

Tags Semânticas - Estrutura

São as tags fundamentais que ajudarão a estruturar um documento HTML5 de maneira semântica, fornecendo significado claro e facilitando a compreensão do conteúdo tanto por humanos quanto por máquinas (Google, IA's).

<!DOCTYPE html>:
Define a versão do HTML que está sendo usada (HTML5).
<html> </html>:
Define o início e o fim do documento HTML.
<head>:
Contém informações sobre o documento, como metadados, título e referências a estilos e scripts.
<title>:
Define o título da página, exibido na barra de título do navegador.
<body> </body>:
Contém o conteúdo principal da página.
<header> </header>:
Representa a introdução ou um cabeçalho da seção (logotipos, títulos, navegação, etc).
<nav> </nav>:
Agrupa os links da navegação principal do site.
<footer> </footer>:
Define o rodapé ou área de conclusão de um documento ou de uma seção.

Tags Semânticas - Conteúdos

<main>:
Define o conteúdo principal da página, excluindo cabeçalhos, rodapés e barras laterais.
<article>:
Representa um conteúdo independente e autossuficiente que pode ser reutilizado e distribuído separadamente, como um post de blog.
<section>:
Agrupa conteúdo tematicamente relacionado dentro de um documento. Pode conter <header> e <footer>.
<aside>:
Representa conteúdo relacionado ao conteúdo ao redor, como barras laterais ou notas laterais.
<h1> a <h6>:
Define títulos de diferentes níveis de importância esua hierarquia. <h1> é o mais importante, e <h6> é o menos importante.
<p>:
Define um parágrafo. Deum parágrafo para o outro mudamos o foco do mesmo assunto.
<a>:
Cria um hyperlink para navegação ou referência a outros recursos/páginas.
<img>:
Insere uma imagem na página.
<ul> , <ol> e <dl>:
Criam listas não ordenadas, ordenadas e de descrição, respectivamente.
<li> , <dt> e <dd>:
Criam itens de listas, item e descrição do item de lista, respectivamente.
<strong>, <em> e <br>:
Enfatiza o texto com negrito, Enfatiza o texto com Italico e quebra de linha.

Paradigmas de Programação

  • Existem diversos paradigmas.
  • Um paradigma é uma forma ou norma para se fazer algo.
  • No PHP podemos usar várias formas, em linguagens como Java apenas a POO
  • Procedural ou Estruturada: Estrutura o código em procedimentos ou funções.
  • Orientado a Objetos: Utiliza classes e objetos para organizar o código.

Programação Orientada à Objetos

  • Pensar em programação orientada a objetos é como organizar um jogo de Lego.
  • Cada bloco (objeto) tem suas próprias peças (dados) e habilidades (funções).
  • Você cria diferentes tipos de blocos (classes) com características específicas, e, quando necessário, os combina para construir algo maior (programa).
  • Isso ajuda a estruturar o código de forma mais modular e compreensível para humanos e máquinas.
  • Nasceu em 1960 e ganhou força com linguagens como JAVA, C#, Python e desde PHP7.

Classes

  • Uma classe em programação é como um modelo ou um plano para criar objetos.
  • Pense nela como um esboço que define as características e comportamentos que os objetos criados a partir dela terão.
  • Possui 04 pilares:
  • Encapsulamento: Consiste em agrupar dados (atributos) e métodos (comportamentos) relacionados em uma unidade chamada classe protegendo/limitando seus acessos.

  • Herança: Permite que uma classe herde características (atributos e métodos) de outra classe. A classe que é herdada é chamada de classe pai ou superclasse, e a classe que herda é chamada de classe filha ou subclasse.
  • Polimorfismo: Permite que objetos de diferentes classes sejam tratados como objetos de uma mesma classe por meio da sobreposição de métodos simplifiando a interação entre classes.

  • Abstração:Envolve a criação de classes e objetos que representam entidades do mundo real, simplificando e abstraindo os detalhes complexos.
            
            
                class Animal {
                    private $nome;
                    private $som;
                
                    public function __construct($nome, $som) {
                        $this->nome = $nome;
                        $this->som = $som;
                    }
                
                    public function fazerSom() {
                        echo "{$this->nome} faz {$this->som}\n";
                    }
                }
                
                // Criando instâncias da classe Animal
                $gato = new Animal("Gato", "miau");
                $cao = new Animal("Cachorro", "auau");
                
                // Chamando método da classe
                $gato->fazerSom();
                $cao->fazerSom();

Tipos de Acesso no PHP

  • Public: são acessíveis de qualquer lugar, dentro ou fora da classe. Isso significa que eles podem ser acessados diretamente, sem restrições.
  • Private: são acessíveis apenas dentro da própria classe. Isso impede que sejam acessados ou modificados por código fora da classe.
  • Protected:são semelhantes aos private, mas com uma exceção: eles também podem ser acessados por classes derivadas (subclasses).

Abstração | Polimorfismo

            
                
                    abstract class Animal {
                        abstract public function emitirSom();
                    }
                    
                    class Cachorro extends Animal {
                        public function emitirSom() {
                            echo "Au Au\n";
                        }
                    }
                    
                    class Gato extends Animal {
                        public function emitirSom() {
                            echo "Miau\n";
                        }
                    }
                    
                    $cachorro = new Cachorro();
                    $gato = new Gato();
                    
                    $cachorro->emitirSom();
                    $gato->emitirSom();   
                
        
            
                
                    interface EmitirSom {
                        public function som();
                    }
                    
                    class Cachorro implements EmitirSom {
                        public function som() {
                            echo "Au Au\n";
                        }
                    }
                    
                    class Pato implements EmitirSom {
                        public function som() {
                            echo "Quack Quack\n";
                        }
                    }
                    
                    $cachorro = new Cachorro();
                    $pato = new Pato();
                    
                    function executarSom(EmitirSom $animal) {
                        $animal->som();
                    }
                    
                    executarSom($cachorro);
                    executarSom($pato);
